## Что реализовано

### Архитектура

Проект разворачивается через **Docker Compose** и состоит из четырёх сервисов:

| Сервис        | Описание                                         |
| ------------- | ------------------------------------------------ |
| `db`          | PostgreSQL 16 с healthcheck                      |
| `migrations`  | Применяет Alembic миграции при старте            |
| `import-data` | Загружает CSV данные в БД (запускается один раз) |
| `fastapi`     | FastAPI приложение                               |
| `nginx`       | Веб-сервер (reverse proxy для FastAPI)           |

Сервисы запускаются в правильном порядке через `depends_on` с условиями `service_healthy` и `service_completed_successfully`.

### API эндпоинты

#### Авторизация

| Метод  | Эндпоинт      | Описание                                      |
| ------ | ------------- | --------------------------------------------- |
| `POST` | `/auth/login` | Авторизация по email/password, возвращает JWT |

Тестовые пользователи логинятся с паролем `user_{id}`, например `user_1`.

#### Пользователи

| Метод | Эндпоинт    | Описание                                             |
| ----- | ----------- | ---------------------------------------------------- |
| `GET` | `/users/`   | Список всех тестовых пользователей (без авторизации) |
| `GET` | `/users/me` | Данные текущего пользователя                         |

#### Лояльность

Все эндпоинты требуют авторизацию. Данные возвращаются для пользователя из токена.

| Метод | Эндпоинт                   | Описание                                                              |
| ----- | -------------------------- | --------------------------------------------------------------------- |
| `GET` | `/loyalty/summary`         | Совокупная лояльность: рубли, мили, баллы Браво и рублёвый эквивалент |
| `GET` | `/loyalty/history`         | Полная история начислений, отсортирована по дате                      |
| `GET` | `/loyalty/history/monthly` | История сгруппированная по месяцам (для графика)                      |
| `GET` | `/loyalty/forecast`        | Прогноз на следующий месяц: среднее за 3 месяца × 1.2                 |

Рублёвый эквивалент считается по формуле: `rub + miles × 2.0 + bravo × 0.5`

#### Офферы

| Метод | Эндпоинт   | Описание                                                                        |
| ----- | ---------- | ------------------------------------------------------------------------------- |
| `GET` | `/offers/` | Акции партнёров и продукты экосистемы, отфильтрованные по сегменту пользователя |

Сортировка: сначала продукты экосистемы (ECOSYSTEM), затем партнёры по убыванию % кэшбэка. Продукты экосистемы скрываются если у пользователя уже есть этот продукт.

#### ИИ-рекомендации

| Метод | Эндпоинт        | Описание                                                      |
| ----- | --------------- | ------------------------------------------------------------- |
| `GET` | `/ai/recommend` | Персонализированный совет через Claude API (claude-haiku-4-5) |

Claude получает данные о накоплениях, прогнозе и доступных акциях пользователя и генерирует короткий персонализированный совет (2-3 предложения).

#### Геймификация

| Метод  | Эндпоинт                     | Описание                                              |
| ------ | ---------------------------- | ----------------------------------------------------- |
| `GET`  | `/gamification/streak`       | Текущий стрик, рекорд и прогресс до следующей награды |
| `POST` | `/gamification/streak/visit` | Трекинг ежедневного визита                            |

Пороги стрика: 3 дня (+20 ₽), 7 дней (+100 ₽), 14 дней (+200 ₽), 30 дней (+500 ₽), 60 дней (+1000 ₽), 100 дней (+3000 ₽).

### CI/CD

GitHub Actions pipeline (`.github/workflows/ci.yml`) запускается на каждый пуш и pull request в `master`.

**Lint** — проверка кода:

- `ruff` — линтер
- `mypy` — статическая типизация

**Test** — автотесты:

- Поднимает PostgreSQL как service
- Запускает 15 pytest тестов
- Покрывает: loyalty API (9 тестов), offers API (6 тестов)

**Deploy** — автодеплой на сервер:

- Подключается через Tailscale + SSH
- Выполняет `git pull` и `docker compose up -d --build`
- Запускается только после успешных lint и test
- Только при пуше в `master`

# Гайд по использованию

## 1. Клонирование репозитория

```bash
git clone https://github.com/repen7ant/LoyaltyHub.git
cd LoyaltyHub
```

## 2. Создание .env

```bash
cp .env.example .env
cd fastapi
cp .env.example .env
```

Заполнить `fastapi/.env`:

| Переменная          | Описание                                                                           |
| ------------------- | ---------------------------------------------------------------------------------- |
| `POSTGRES_USER`     | Пользователь БД                                                                    |
| `POSTGRES_PASSWORD` | Пароль БД                                                                          |
| `POSTGRES_HOST`     | Хост БД (внутри Docker: `db`)                                                      |
| `POSTGRES_PORT`     | Порт БД (по умолчанию `5432`)                                                      |
| `POSTGRES_DB`       | Имя БД                                                                             |
| `SECRET_KEY`        | Секрет для JWT — сгенерируй случайную строку                                       |
| `ANTHROPIC_API_KEY` | API ключ Claude — получи на [console.anthropic.com](https://console.anthropic.com) |

Сгенерировать `SECRET_KEY`:

```bash
openssl rand -hex 32
```

## 3. Запуск

```bash
docker compose up -d --build
```

## 4. Разработка

После запуска API будет доступно по адресу:

```bash
http://localhost:8000
# swagger:
http://localhost:8000/docs
```

Логи бэкенда:

```bash
docker compose logs fastapi -f
```

Логи всех контейнеров:

```bash
docker compose logs -f
```

## 5. Остановка проекта

```bash
docker compose down
```

С удалением данных:

```bash
docker compose down -v
```
